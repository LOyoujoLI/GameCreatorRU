## Start

The **loop** directive is a lazy tool that can be used to quickly create repetitive Command when we have to write them.

So I suggest that when you write repetitive, overly familiar Command, you can think about whether you can use loops to simplify them, so that they are convenient and conducive to later maintenance.

## Caution

### Waiting

The loop must be with wait, otherwise it will be stuck, the wait does not mean wait Command, but Command with wait characteristics.

Of course, if you don't understand it, you can also use the wait Command to wait for 1 frame directly.

![](img/In-depth/009/001.png)

### Account for processes

Many people have problems with process occupancy when using loops.

Since Command are executed in order from top to bottom, and occupying a process we can interpret as a queue being queued up, and the loop keeps getting in front of the queue causing the later Command to not be executed.

![](img/Understand/004/001.png=400-)

In this case we need to use independent events, there is a checkbox for independent events in the Common Event or event page, we can check it.

![](img/Understand/004/003.png)

![](img/Understand/004/002.png=420-)

### Interrupting the loop

When we don't want the loop to continue, we can use the **Break Loop** Command. Note that the interrupt loop Command must be inside the loop Command you wish to interrupt, so most of the time it is used in conjunction with Conditional Branch.
