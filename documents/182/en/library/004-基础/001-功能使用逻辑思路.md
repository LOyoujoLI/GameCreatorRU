## Start

Inevitably, when making a game we want to do a certain function but are not sure how to start, so here are the main ideas about function calls.

## Standing avatar

We can find the **Standing avatar** module in the **Database** and add the corresponding material directly.

The subsequent templates related to standing avatar will be called through the standing avatar module.

![](img/Basic/001/001.png=600-)

## Animation

We can find the **Animation** module in the **Database** and just make it as needed afterwards.

Subsequent templates related to animation are called through the animation module.

![](img/Basic/001/002.png=600-)

## Dialog box styles

We can find the **Dialog Styles** module in the **Database** and replace the corresponding material directly.

If there are individual components you don't want to display, you can drag them off-screen or set the transparency to 0.

The styles called by the displayed dialogs are based on this.

![](img/Basic/001/003.png=600-)

## Game interface

We can find the **Interface** module in the **Database** and directly add the corresponding components as well as materials.

All the interfaces in the game can be made by the interface module, such as the title interface.

![](img/Basic/001/004.png=600-)

## CG howcase

We can find the **CG howcase** module in the **Database** and add the material directly.

For more information on how to use CG howcase, please refer to [How to add CG howcase](?file=003-入门/006-如何添加鉴赏CG)

![](img/Basic/001/005.png=600-)

## Get the content of the game

There is a game variable in variables, and we can get the content of the game through the game variable.

You can check [Knowing game variables](?file=004-基础/006-认识游戏变量)

## Game system process

Starting the game, archiving, and returning to the title screen are all controlled by Command.
