## Start

Variables we can understand as a kind of container for storing values, like the unknowns we use when learning about numbers.

Variables are dynamic values that are whatever we want them to be.

## Types

We know that variables can store values, so there are certain classifications of variables in GC.

### Numeric variables

Numeric variables are variables that specifically store numbers, and we can usually use numeric variables to perform mathematical formulas.

The default value of a numeric variable is 0.

![](img/Basic/002/001.png)

![](img/Basic/002/002.png)

### Switch variables

Switch variables have only two states, ON(on)/OFF(off).

Usually we can use the switch variable to make a judgment whether the game has something or not.

The default value of the switch variable is OFF (off).

![](img/Basic/002/003.png)

![](img/Basic/002/004.png)

### String variables

String variables can store all characters, such as Chinese as well as special symbols.

We can make functions such as player-defined names through string variables.

The default value of string variables is empty (i.e. nothing is displayed).

![](img/Basic/002/005.png)

![](img/Basic/002/006.png)

## Ng+ variables

Ng+ variables are different from normal variables, where the values stored in normal variables are valid only in one game archive, while Ng+ variables are global and have an effect outside the game (without executing the start game).

Usually, we can make different content judgments that appear when the player plays multiple games.

If you want to use Ng+ variables, remember to execute the Save command.

![](img/Basic/002/007.png)

## Game variables

Game variables are different from other variables in that they exist to get the value of the game content, so we can use them if we want to get some content in the game.

Game variables are not separate Command, but can usually be found on the corresponding variable Command, Conditional Branch, and interface components.
