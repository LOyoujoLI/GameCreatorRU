# Status page and appearance conditions
The status page is used in conjunction with the occurrence conditions, and when the occurrence conditions of the status page are met, it is switched to that status page (in descending order),<br>
The same scene object can have multiple state pages, i.e., with multiple different properties and events.<br>
The fact that an object can be rendered in different states depending on different conditions is a common tool within game production.<br><br>

## Conditions of appearance
The status page will appear when the conditions for the bar are met.<br>
If more than one status page is satisfied at the same time, the priority is from bottom to top.<br>
As long as there is a state page that satisfies the condition, the object is considered== to be on the scene==, otherwise it is considered== to be removed== from the scene.<br>
==Occurrence conditions allow multiple configurations, and all must be satisfied before they are allowed to occur.==<br>
Variable details can be found in [Event System - Variables](?file=003-GC本体文档/0090-事件系统/002-变量)。<br>
