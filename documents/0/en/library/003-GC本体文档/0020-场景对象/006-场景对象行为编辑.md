## Does not block the current event page execution
Behavior internal if there is a need to wait for related instructions only internal independent wait, and does not affect the execution of subsequent instructions on the current event page, <br>for example, an object is dispatched to a group of behaviors and the object will perform those behaviors on its own, can be dispatched to multiple objects at the same time and they can go to the same or different groups of behaviors at the same time.<br>
If you need to wait for the execution of the behavior instruction to complete before the current instruction subsequent events can be used [Behavior instruction: wait for the behavior to end](#Behavior instruction: wait for the behavior to end).

## Behavior command: change avatar
The command to change the avatar is provided by the GC base layer. After using this command, select the avatar, orientation, action, and frame to specify that the change is completed.<br>
The command is done instantly, without any waiting time, and if the avatar is changed several times in succession, he always shows the last one,<br>
The effect of gradual change can be achieved only when there is a "wait time" command in the middle.<br>
<img src="img/2/场景对象/3-8.png" /><br>

## Behavior instruction: wait for the behavior to end
After all the actions sent out before the [Execute Waiting for End of Actions] instruction on the current event page are executed, the next instructions are executed.<br>
If a circular behavior is dispatched, it will never wait for it to end.

## Customized Behavior
Allow users to customize the editing behavior, and the edited behavior can also be previewed in real time in the preview window. Refer to [Custom Editor - Custom Behavior](?file=003-GC本体文档/1010-自定义编辑器/035-自定义行为) for details。<br>

==Since GameCreator allows users to make their own behaviors, the specific behavioral features are determined by the specific project.==
