# Scene Object
A scene object is an object placed on a scene, and each scene object has a unique [Scene object prototype](#Advanced Features: Scene Object Prototype)<br>
The scene object acts as a display container and may contain a avatar, a mounted interface (e.g. blood bars, names, etc.), mounted animations<br>
The scene object may contain several [Scene Object Module](#Scene Object Module)。<br>

## Regarding the base attributes:
Scene objects generally have basic attributes such as coordinates and display levels, while display objects with avatar have basic attributes such as body type, hue, and playback frequency for avatar.<br>In addition, there is a secondary toolbar on the top of the header after selecting the scene object, which allows you to set the name, quickly change the face, action and avatar.<br>
<img src="img/new/2-26.png" /><br>

## About Triggers:
Support binding custom triggers, such as click on the object event, touch the object event, etc., customize a trigger category name and implement the trigger in the script.<br>
Multiple events can be executed simultaneously if the execution conditions are met.<br>
>For example, each object in the "Feather of cyan" template has: "click event", "touch event", "parallel event", etc.

## Regarding the emergence conditions and status pages:
The status page is used in conjunction with the appearance conditions, and is switched to when the appearance conditions of the status page are met (bottom-up order).<br>
Multiple state pages can be preset for the same scene object, i.e., it has several different properties and events, but only one state page is currently displayed.<br>
The fact that an object can be rendered in different states depending on different conditions is a common tool within game production.<br><br>
For example, the same scene object appears in different states due to different conditions, i.e.<br>

## Regarding scene object behavior:
GC supports customizing the behavior of scene objects, usually the templates are preset with relevant object behavior for users to edit the object behavior through the behavior editor.<br>
Custom object behaviors appended by the author or template creator can also be previewed in real time to the playback effect.<br>
Each time a set of behaviors is added to an object, it is added to a queue that has a last-added-first feature, i.e., it is added to the queue instead of being executed first, and the group of behaviors in the previous queue is executed only after the execution is completed:<br>
> For example, the default behavior of a scene object is: ABC, when it finishes executing A, then add him to the behavior group of DEF again, then its actual order is A-D-E-F-B-C.

For example, behavior in RPG games may include moving, facing, jumping, etc.<br>
Allow authors to edit in the default behavior of an object or specify in an event which behavior an object performs, while supporting live previews of:<br>

## Advanced Features: Scene Object Prototype
A scene object prototype is a predefined class of objects consisting of ==public modules== and ==pre-installed scene object modules==.<br>
You can understand that different archetypes are different species or categories<br>
Items that are usually edited for senior developers, and that ordinary creators only need to know about.<br>
### Public Modules
A public module is a common feature shared by all classes of prototypes, all having common characteristics, consisting of: ==properties==, ==display objects==, ==bound scripts==, ==editor preview==<br>

### Scene Object Module
Scene object modules are assemblable components. When a scene object has a module, it has the characteristics of that module.<br>
Modules usually come from template pre-built, plug-in malls or senior developer homebrew.<br>

## Advanced features: custom display objects.
The scene object can use a avatar, and several ==animation objects== and ==interface objects==, all of which can be preset in the prototype.<br>
You can specify a fixed animation, interface, or a class of animations or interfaces, which allows the author to select each object in the property bar.<br>
As for what you define these display objects to do, you currently need to implement the relevant functionality in the script.<br>
> For example, the scene object has a blood bar, which is related to the custom property - life value, and the current state of the blood bar screen is determined according to the current life value and maximum life value<br>
> Usually, the templates are pre-set with these things so that they can be used directly by regular authors. Advanced developers, on the other hand, can modify and add or delete at will.
