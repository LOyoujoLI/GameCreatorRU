## Basic Properties
<table>
 <tr>
    <td><b>Properties</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>x</td>  <td>Horizontal coordinates of the scene object relative to the scene</td>
 </tr>
 <tr>
    <td>y</td>
    <td>
      Vertical coordinates of the scene object relative to the scene
    </td>
 </tr>
 <tr>
    <td>Levels</td>
    <td>
      Present shading relationships according to different layer settings<br>
      1.Bottom layer display: lower than the middle layer, in the same layer will automatically swap the shading relationship according to the y coordinate<br>
      2.Intermediate layer display: in the same layer will automatically swap the shading relationship according to the y coordinate<br>
      3.The most front display: above all layers, in the same layer will automatically swap the shading relationship according to the y coordinate
    </td>
 </tr>
 </table>

## Avatar properties
<table>
 <tr>
    <td><b>Properties</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Auto Play Action</td>
    <td>
      Automatic cycling of the sequence of frames according to the current direction and movement and frame rate<br>
      The current frame map of the avatar is determined by: Avatar Style => Action => Orientation => Number of frames
    </td>
 </tr>
 <tr>
    <td>figure (body)</td>
    <td>
      Scaling rate of the avatar, scaled by the avatar crosshair (usually at the foot of the avatar)
    </td>
 </tr>
 <tr>
    <td>Alpha</td> <td>Alpha of avatar</td>
 </tr>
 <tr>
    <td>Hue</td>
    <td>
    Take the value range -180 ~ 180, avatar uniform tone rotation.<br>
    If you wish to change the color tone locally, you can refer to [avatar settings](?file=003-GC本体文档/0030-行走图/004-编辑行走图)
    </td>
 </tr>
 <tr>
    <td>Playback frequency</td>
    <td>
    This property affects the speed of playback in frames/second if the action is in autoplay
    </td>
 </tr>
 </table>

## Show List
The display list may contain avatars, animations, and interfaces. The avatar is unique in that only one can be mounted, while the other display objects allow multiple ones.<br>
How many mounted display objects a scene object has is determined by its prototype.<br>
==Prototypes are specifically free to set the display list, usually pre-defined by the senior producer==, Reference [Scene object prototype](?file=003-GC本体文档/1010-自定义编辑器/020-场景对象原型)。<br>
In addition plugins can additionally add display lists, such as[《Plugin: Header Display Name》](http://www.gamecreator.com.cn/plug.php?id=85)。<br>

## Custom Properties
Which custom properties a scene object has is determined by the public properties, additional object modules.<br>
==Prototype specific custom properties can be freely set, usually pre-defined by the senior producer==, Reference [Scene object prototype](?file=003-GC本体文档/1010-自定义编辑器/020-场景对象原型)。<br>
Usually the template will be prefabricated with some properties that are expected of the scene objects in that template, such as movement speed.<br>
