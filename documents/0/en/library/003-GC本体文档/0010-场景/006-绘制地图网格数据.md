The grid data layer is used for in-game logic operations.<br>
After new creation all scenes and block sources have this grid data layer, not just for the current scene, and can be painted with this data.<br>
== Usually a template preset is sufficient, and advanced developers can add, delete or modify grid data layers by setting them here and implementing them in scripts. ==<br>
== The grid data layer does not allow changing positions and is always referenced by position (starting from position 0). ==<br><br>

For example, the pre-made barrier layer, mask layer, and bridge attribute layer in the "Feather of cyan" template.<br>

## Create new grid data layer
Select the grid data layer (or the rightmost layer if there is none) and click the right mouse button to bring up the menu and select <u>Create new grid data layer</u>。<br>
<img src="img/2/场景/6-2.png" /><br>
<table>
 <tr>
    <td><b>Layer Properties</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Name</td>  <td>Display name of this grid data layer</td>
 </tr>
 <tr>
    <td>Drawing colors</td>  <td>Painted colors, used only by game makers for reference to distinguish different data</td>
 </tr>
 <tr>
    <td>presentation</td>  <td>Explanatory hint when hovering over this grid data layer</td>
 </tr>
 </table>

## Operation
Switch to this grid data layer and start painting the data
<table>
 <tr>
    <td><b>Operation</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Open</td>  
    <td>
      ==Single block painting==: Left mouse button click on the block source or press to start painting<br>
      ==Area Painting==: Right mouse button to select an area for painting
    </td>
 </tr>
 <tr>
    <td>Close</td>
    <td>
      == Single block cancellation settings==: SHIFT+left mouse button click on the block source or press and start painting type of cancellation settings<br>
      ==Area cancellation settings==: SHIFT+right mouse button to frame an area for paint-style cancellation settings
    </td>
 </tr>

 </table>
