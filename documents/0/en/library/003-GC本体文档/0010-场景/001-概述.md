Scenes contain == scene layers == display, placement == scene objects == and == scene related trigger events ==.<br>

## Grid support:
After setting the grid pixel size in the settings (Menu - Settings - Scene Module), the pixel size of each grid is fixed to this grid pixel size (e.g. 48x48 or 32x32).<br>
The grid is generally used in the editor to draw maps and data information, etc., while in the actual gameplay implementation it is up to the developer to choose to refer to this property or not.<br>
Some game templates do not need grid and usually turn off the grid function (Menu - Custom Editor - General Settings - System Features - Grid Display).<br><br>

## Scene layers:
The layers are divided into: ==Block layer==, ==Picture layer==, ==Grid data layer==.<br>
There is no upper limit on the number of layers, so you can add or remove them freely, but the more layers you use, the more performance and memory space they consume.<br>
Layers can be set with properties such as offset, loop, auto-move (e.g. to create fog effects), telephoto scale (e.g. to create multiple telephoto effects), etc.<br>
Block materials can also be set up with a grid data layer for reuse: for example, if you paint barrier data on a tree root, it has the properties of a barrier anywhere it is placed on the scene pair (the barrier data layer is a custom data layer).<br>

## Placing and finding scene objects:
Scene objects can be placed on top of the scene, such as as NPCs or decorations or even interfaces placed on the scene, etc.（Concept details can be found in [Scene Object](?file=003-GC本体文档/0020-场景对象/001-概述)）<br>
At the same time the map can use the scene list to select the corresponding scene object, according to the name or ID of the object to query, double-click to lock to this object, easy to quickly find the desired object<br><br>

## Scene custom trigger events support:
Support binding custom trigger event categories, such as the event of entering the scene, the event of leaving the scene, etc.<br>
For example, the "Feather of cyan" template has a trigger event called "Enter Scene".<br>
Advanced developers can write custom trigger event categories (menu - custom editor - bound event pages - event pages bound to scenes, and write script programs to implement them)<br>
<img src="img/new/2-4.png" /><br>

## Regarding the server-side scenario of the network kernel:
In the case of games or game templates created with GameCreator2DNet:<br>
The server's scenario acts as the logical scene and generally manages all the scenario objects, players, etc. under the scenario.<br>
The scenes on the server side are created due to the engine, and all scenes and the scene objects on the scenes are created during the engine initialization.<br>
==The following use cases are available for reference:==<br>
For example, MMOARPGs need to execute the logic of all monsters in the scene (e.g. monitoring whether there are players around the monsters, chasing players, etc.).<br>
For example, after entering a battle in card battles players are in a copy of the card battle scenario, which exists only for players who are playing or watching the battle.<br>
For example, in MOBA type games, after the match is completed, let all players enter the scene copy and start to execute the frame synchronization logic, etc.<br>
