## New Map
Click the + sign in the map list (①).<br>
<img src="img/new/2-6.png" /><br>
If a grid size setting exists it means that the project is enabled for grid display. The pixel size of the grid is uniform and can be set in the editor settings.<br>
The grid size and pixel size are interrelated, the editor will automatically adjust the size of the other one after modifying one of them, and it is better to set the pixel size as an integer multiple of the grid size.<br>
<img src="img/new/2-7.png" /><br>
Clicking OK creates a new map within the current map file.<br>

## Resize the map
The map can be resized for existing maps, and if the map is made smaller, the drawn map and scene objects within the reduced portion will be eliminated.<br>
You can also resize the map or offset the drawn map within the currently selected scene by using "Expand and Offset" in the Scene Properties panel.<br>
==This operation cannot be withdrawn, so please operate with caution. If you make a mistake, you can simply close the project without saving it and restart it again.==
<table>
 <tr>
    <td><b>Operation</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Menu Operations</td>  <td>
      Right mouse click on the map file and select<u>Properties</u>: Modify related dimensions<br>
    </td>
 </tr>
 </table>

## Copy Map
Copy and paste the map to create another copy of the previously copied map under the specified map folder.<br>
Except the map ID is different, the layer settings, scene properties, scene objects, related events, etc. inside are all the same as the original map.<br>
You need to select a map file with the mouse before you can use the copy function.
<table>
 <tr>
    <td><b>Operation</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Keyboard operation</td>  <td>
      CTRL+C Copy CTRL+V Paste<br>
    </td>
 </tr>
 <tr>
    <td>Menu Operations</td>  <td>
      Right mouse click on the map file and select <u>Copy</u>,Then right-click on the location where you want to paste and select: <u>Paste</u><br>
    </td>
 </tr>
 </table>

## Map Status Settings
These settings are limited to editor use and are not relevant when the game is running.
<table>
 <tr>
    <td><b>Operation</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Display Grid</td>  <td>
      Checking this box will display a black grid in the scene view, and the grid size will be determined by the preset grid pixel size in the editor settings<br>
      Shortcut keys:CTRL+H
    </td>
 </tr>
 <tr>
    <td>Object alignment grid</td>  <td>
      Check this box to align the center of the grid when dragging and dropping scene objects<br>
      If this is not checked, the grid can be aligned while holding down SHIFT while dragging scene objects
    </td>
 </tr>
 <tr>
    <td>Non-current layer display mode</td>  <td>
      <table>
       <tr>
          <td>Normal</td><td>Unchecked layers are displayed normally, check this to preview the actual effect when editing layers, but it is not easy to distinguish the currently selected layer</td>
       </tr>
       <tr><td>Dull display</td><td>Unselected layers are shown dimly</td></tr>
       <tr><td>No display</td><td>Unselected layers are not displayed</td></tr>
      </table>
    </td>

 </tr>

 <tr><td>Grid state layer settings</td><td>Checking the specified grid data layer will display it at all times</td></tr>

 <tr><td>Show scene dynamic effects</td><td>Check this box to get a real-time preview of the motion of the layer and the dynamic effects of the scene object's motion playback</td></tr>
 </table>

## Scene Basic Properties
<table>
 <tr>
    <td><b>Properties</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Map Size</td>  <td>
      Shows the pixel dimensions of the map, and also shows the grid dimensions when the grid display is turned on<br>
    </td>
 </tr>
 <tr>
    <td>Extensions and Offsets</td>  <td>
      Expanding or deleting map dimensions for the current map and offsetting drawn blocks (truncating them if out of range)
    </td>
 </tr>
 <tr>
    <td>Client</td>  <td>
      Client-side scenario binding class, the script implementation class used to actually run the scenario logic, different maps support different binding classes<br>
      Default map creation is based on the 「Default Client Binding Class for New Scenes」 pre-set in the editor settings
    </td>
 </tr>
 <tr>
    <td>Server Side</td>  <td>
      [Network version only] Server-side scene binding class, the script implementation class for the server to actually run the logic of the scene, different maps support different binding classes<br>
      Default map creation is set according to the 「Default server binding class for new scenes」 pre-set in the editor settings
    </td>
 </tr>
 <tr>
    <td>BGM</td>  <td>
      Background music that plays automatically when you enter the scene<br>
      This property is usually implemented by the project layer, e.g. some fade in and fade out effects may be added
    </td>
 </tr>
 <tr>
    <td>BGS</td>  <td>
      Ambient sound effects that play automatically when you enter the scene<br>
      This property is usually implemented by the project layer, e.g. some fade in and fade out effects may be added
    </td>
 </tr>
 <tr>
    <td>Preloaded Map Resources</td>  <td>
      Checking this will preload the image resources involved in the layers of the scene when the scene is preloaded<br>
      Includes block resources for the block layer and image resources for the image layer<br>

 </tr>
 <tr>
