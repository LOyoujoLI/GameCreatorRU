The layers are divided into display layers and grid data layers, where the display layers include block layers, picture layers and object layers, and the actual display levels of the layers are determined by the display order from left to right and from low to high.<br>
==Note: The grid data layer is a logical data layer used for game logic operations to obtain the corresponding grid data layer in the order of display.==<br>
> The following figure, 0 = barrier layer 1 = mask layer 2 = bridge attribute layer, once the location change or deletion may lead to drawing mesh data and the actual effect does not match

## Create a new picture layer
Right-click on the layer and select <u>New construction</u> => <u> Image Layer </u>。<br>
After creating a new layer, you can modify the properties of that layer in the Layer Properties panel on the right.<br>
<table>
 <tr>
    <td><b>Layer Properties</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Edit Mode</td>  <td>You can freely edit the position, size and rotation of the image in the scene after turning on the editing mode<br>
     Non-editing mode only allows setting properties that cannot be edited but can be previewed in real time (e.g. tiling)<br>
     The default switch to this layer is in edit mode, but you can also choose to lock it in non-edit mode (when locked, it goes into non-edit mode by default)</td>
 </tr>
 <tr>
    <td>Pictures</td>  <td>Select a large image to display as the background for this layer</td>
 </tr>
 <tr>
    <td>Offset</td>  <td>The coordinate position offset of the layer, including the horizontal coordinate x and vertical coordinate y</td>
 </tr>
 <tr>
    <td>Scaling</td>  <td>Scaling of layers, including horizontal scaling sx and vertical scaling sy</td>
 </tr>
 <tr>
    <td>Tilt</td>  <td>Tilt of the layer, including horizontal scaling skewX and vertical scaling skewY</td>

 <tr>
    <td>Flattening</td>  <td>Tiling of layers, check this to cycle through the layers, including x-direction tiling and y-direction tiling</td>

## Create a new block layer
Right-click on the layer and select <u> New </u> => <u> Block layer </u>。<br>
The properties are the same as for the new image layer (except for the image settings).

## Create new grid data layer
The grid data layer is used for in-game logic operations.<br>
After new creation all scenes and block sources have this grid data layer, not just for the current scene, and can be painted with this data.<br>
==Usually a template preset can be used, and advanced developers can add, delete or modify the grid data layers by setting them up here and in scripts==。<br>
==The grid data layer does not allow changing positions and is always referenced by subscripts (starting from 0)==。
<table>
 <tr>
    <td><b>Layer Properties</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>name</td>  <td>Display name of this grid data layer</td>
 </tr>
 <tr>
    <td>Drawing colors</td>  <td>Painted colors, used only by game makers for reference to distinguish different data</td>
 </tr>
 <tr>
    <td>presentation</td>  <td>Explanatory hint when hovering over this grid data layer</td>
 </tr>
 </table>
