# Event System Overview
The event system is a visual programming system. The game content and logic is written through visual commands. <br> 
The execution of event page content is generally triggered when conditions are met, such as a pre-defined click event for an interface component that needs to be triggered when the player clicks on that component. <br> 
Event commands are executed one by one in top-to-bottom order, and are subject to logic control resulting in line skipping execution or jumping back to the specified line for execution,<br> 
and there are also commands that need to wait for execution to complete before the next line is executed. <br>

### Trigger Line (Trigger)
Each event execution corresponds to one trigger line, and multiple events executing at the same time means multiple trigger lines are used. <br> 
The custom trigger event category supports setting multiple trigger lines or a single trigger line. If it is a multiple trigger line, a new trigger line will be created for each execution to execute the content of that event page. <br>
> For example, let's say you create an interface component that clicks on an event - the execution takes 10 seconds to complete.<br> 
If the interface component's click event is a single trigger line, the player clicks once to start the event, and then clicks again to invalidate it, and only waits 10 seconds for the execution to complete before it can continue. <br>
And if it's a multi-trigger line, the player clicks once to start a new trigger line, and each line takes about 10 seconds to execute.

### Trigger events for scenes
Support for binding custom trigger event categories, such as events that enter the scene, events that leave the scene, and so on, with a custom trigger category name that can be triggered in the script. <br> 
For example, the "ARPG - Book of black" template has a trigger event named "Enter Scene". <br>

### Trigger events for scene objects
Support binding custom trigger event categories, such as click on object events, touch object events, etc., customize a trigger category name and implement the trigger in the script. <br>Multiple events can be executed simultaneously if the execution conditions are met. <br>
>For example, the "ARPG - Book of black" template has 3 types of events: "Click event", "Touch event", and "Parallel event"

<img src="img/t24.png" />

### Trigger events for interface components
It supports binding custom trigger event categories, such as click events, mouse hover events, etc., with a custom trigger category name and then implementing the trigger in the script. <br>Multiple events can be executed simultaneously if the execution conditions are met. <br>
>For example, the "ARPG - Book of black" template has 3 types of events: "click event", "mouse hover event", and "mouseover event"<br>

### Calling events from the event library
The events of the event library are pre-defined event pages that are typically stored as events of the event library when some events or logic will be invoked in multiple places so that they can be invoked everywhere. <br> 
Calling a library event in an event is equivalent to temporarily appending all the events in the library event and waiting for the content of that event to finish before continuing to execute subsequent events. <br>
>For example, if the event execution ABCD, where B is an event from the event library and the content is 1234, then the execution is actually A-1-2-3-4-B-C-D.

<img src="img/t55.png" />

### Calling independent events
Essentially, it starts a new trigger line, independent of this trigger line, and the new trigger line goes on to execute events from the event library independently, while this trigger line continues down the line. <br> The lifecycle of a standalone event ends when the content of the event has been executed (cross-scene execution is supported).
>For example, if the event execution ABCD, where B is an independent event and the content is 1234, then the current trigger line execution is ACD, which starts a second trigger line to execute 1234 at the same time.

<img src="img/t56.png" />


### Calling event fragments and standalone event fragments
Event snippets can be thought of as lightweight events for an event library, equivalent to packing a small insert of events into an event page. <br> 
Standalone event snippets have the same effect as events from a standalone event library<br>
