# System command: Display options
Options is a common form of presentation within the game. This command allows the screen to display options and call different subsequent event commands depending on the options.

## Displayed alongside the dialogue text
If the previous command of the option is dialog text and the dialog style is the same, the previous dialog text does not disappear when the option appears.

## Set default options
When the focus is on the specified option, the image of the preset option focus in this dialog style is displayed. <br>
If the default option is set, the focus will check the preset default option by default. <br>
Right mouse click on the option to check the <u>default option</u>.

## Setting the occasion of cancellation
Right mouse click on the option to check the <u>canceled occasion</u>.
Occasions that are set to be canceled can be canceled by using the ESC key (the key can be modified, refer to [Support keyboard operation](#Support keyboard operation).

## Support keyboard operation
Options is a list component that also supports keyboard operation when list keyboard operation is enabled (UIList.KEY_BOARD_ENABLED). <br> 
The default is to use the arrow keys to move the option, the space bar to confirm, and the ESC key to cancel (if the occasion to cancel is set). <br> 
Changeable key settings: provided by default in [UIList.api](?file=006-API-2D单机内核/002-客户端/003-界面/019-UIList), some project layers may implement visual key change or in-game key change.

## Waiting for events submitted by players
The appearance of this event suspends the execution of subsequent event commands until the player submits them and then continues.

## Blocking events
This event is a blocking event. Blocking events are only allowed one at a time within the game, and if multiple triggers have blocking events, they will take turns waiting for the player to submit them.<br> 
Blocking events must be [Waiting for events submitted by players](?file=003-GC本体文档/0090-事件系统/030-系统指令-等待玩家提交信息).
