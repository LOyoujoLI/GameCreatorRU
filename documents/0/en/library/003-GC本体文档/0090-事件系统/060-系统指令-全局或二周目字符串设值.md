# Global or 2-week string set value
Standalone - Second Week Variables: Variables that are not related to the archive and are loaded when the program is started. This variable will be included for saving when the save global data is called. <br> 
Network version - Global variable: A public variable for the whole game (service), which can be understood as a game world variable. <br> 
A variable that records text.

## Dynamic variable symbols
Dynamic symbols are used to display some variable information, not as fixed characters, but as determined by the actual current value at runtime.
<table>
 <tr>
    <td><b>Dynamic variable symbols</b></td>  <td><b>Description</b></td>
 </tr>
 <tr>
    <td>Global numeric variables/dialogue numeric variables</td>  <td>==[$v1]== 1 means variable number 1</td>
 </tr>
 <tr>
    <td>Global String Variables / Two-Week String Variables</td>
    <td>
      ==[$s1]== 1 means string variable number 1
    </td>
 </tr>
 <tr>
    <td>Numeric variables</td>
    <td>
    ==[@v1]== 1 indicates the numerical variable number 1
    </td>
 </tr>
 <tr>
    <td>String Variables</td>
    <td>
    ==[@s1]== 1 means string variable number 1
    </td>
 </tr>
 <tr>
    <td>Common Functions</td>
    <td>
    ==[@f1]== 1 means the 1st common function
    </td>
 </tr>
 <tr>
    <td>Player input value</td>
    <td>
    ==[@p0]== 0 indicates the input value submitted by player 0. Players can submit information in the form of an array, and 0 indicates the subscript 0, i.e. the value information in the first position
    </td>
 </tr>
 </table>
