## Concept
 <table>
  <tr>
     <td><b>Keyword</b></td> <td><b>Description</b></td>
  </tr>
  <tr>
     <td>Behavior</td> <td>GameCreator allows scene objects to execute a specified set of behavior commands, allowing the object to perform the behavior according to that command</td>
  </tr>
  <tr>
     <td>Behavioral layer</td>
     <td>
       Each set of behaviors added to the object is treated as a new layer, and only after the layer is executed will it return to the previous layer to continue executing the behavior (so the looped behavior cannot return to the previous layer to be executed)<br>If the object is executing the behavior and needs to wait, it can only execute the next behavior after it is done waiting, and if there is a new behavior layer when executing the next behavior,<br>it will start executing the new behavior layer ( For example, the "Move" and "Wait" command of "RPG - Feather of cyan" are both commands that require waiting, so if the object is moving <br>
       and then adds a new behavior group he will not execute it immediately, but will have to wait for this move to finish, and if it needs to be executed immediately, you can check the "Force Stop the Executing Behavior" box)
     </td>
  </tr>
  </table>

## Not blocking outside event page execution
If there are commands inside the behavior that need to be waited for related commands are only waited for internally independently and do not affect the execution of subsequent commands on the outside event page. <br> 
If you need to wait for the behavior command to finish executing, the event following the outside command can be used with [Behavior command: Wait for behavior to end](#Behavior command: Wait for behavior to end).

## Behavior Command: Change avatar
The command to change the avatar is provided by GC's bottom layer, after using this command, select the avatar, orientation, action, and frame to specify the completion of the change. <br> 
This command is instantaneous, without any wait time. If you change the avatar multiple times in a row, it will always be the last one displayed. <br>
<img src="img/2/场景对象/3-8.png" /><br>

## Behavior command: Wait for behavior to end
All actions dispatched before the [wait for end of action] instruction on the current event page are executed before continuing with the next instructions. <br> 
If a cyclic behavior is dispatched, it will never wait for it to finish.

## CustomBehavior
Allow users to customize the editing behavior, and the edited behavior can also be previewed in real time within the preview window. Refer to [custom-editor-custom-behavior](?file=003-GC本体文档/1010-自定义编辑器/040-自定义行为) for details. <br>

==Since GameCreator allows users to make their own behaviors, the specific behavioral features are determined by the specific project.==
