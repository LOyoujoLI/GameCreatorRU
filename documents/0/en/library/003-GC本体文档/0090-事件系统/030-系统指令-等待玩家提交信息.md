# System command: waiting for players to submit information
When this command appears, it will wait for the player to submit it. How and what information the player submits will be determined by the specific project.<br>


## Waiting for events submitted by players
The appearance of this event suspends the execution of subsequent event commands until the player submits them and then continues.

## Blocking events
This event is a blocking event. Blocking events are only allowed one at a time within the game, and if multiple triggers have blocking events, they will take turns waiting for the player to submit them.<br> Blocking events must be [Waiting for events submitted by players](#Waiting for events submitted by players).


## Submit using client-side script
GameCommand.inputMessageAndContinueExecute(Array of submitted messages)<br>
Since this instruction is a blocking event, there is no need to specify the triggerLineID. use the default value.<br>
==The standalone version allows submission of arbitrary values, the web version only allows submission of data that can be JSONized.== <br>
Reference API:
>/\*\*<br>
> \* Enter the information and continue the execution, the event page of the waiting player input, will receive the input parameters.<br>
> \* Can create such as input name, password, QTE system, etc.<br>
> \* Event page production process:<br>
> \*  -- For example, by calling the script: inputMessageAndContinueExecute([123]);<br>
> \*  -- Event: Waiting for player input<br>
> \*  -- Events: can be received by assigning a value to a variable = player input or by displaying the player input in text, etc.<br>
> \* @param inputMessage [Optional] Default value = null Input information<br>
> \* @param force [Optional] Default=false Whether to force the mode (non-forced mode only sends when the client determines that input is needed)<br>
> \* @param delayFrame [Optional] Default=1 Delay how many frames to send<br>
> \* @param triggerLineID [Optional] Default = -1 means submit the message to the specified trigger line, -1 means the default is GameCommand.inputTriggerLine, which is the current trigger line that the system is waiting for player input<br>
> \*/<br>
>static inputMessageAndContinueExecute(inputMessage: any[] = null, force: boolean = false, delayFrame: number = 1, triggerLineID: number = -1): void {
>
>}<br>

## Receive player submissions using scripts
Standalone version: received using client-side script<br> 
Network version: received using server-side script<br
Use trigger.inputMessage in the client script to get an array of messages submitted by the player. <br>
