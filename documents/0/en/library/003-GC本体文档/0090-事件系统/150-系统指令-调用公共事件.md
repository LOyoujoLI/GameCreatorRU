# Calling events from the event library
An event library is a pre-made page of stored events that can be called from multiple locations once it is made. <br>
After calling an event from an event library, you need to wait for the event to finish executing internally before continuing to execute subsequent instructions (except for standalone events).

## Independent Events
Creating a standalone trigger to execute events from an event library does not block the execution of the current trigger. <br> The trigger and executor of the newly generated standalone trigger are the same as the currently invoked event page. <br> 
A standalone event is not destroyed across scenes if both the trigger and the executor of the standalone event are the player's scene objects.
