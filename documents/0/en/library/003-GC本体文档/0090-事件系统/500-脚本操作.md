# Code example: manipulate variable
The variable system is a pre-defined array of data that the system provides in the form of ==global variables (two-week variables)== and ==player variables==. <br> 
And each variable is divided by variable type: ==Numeric variables==, ==String variables==, and ==Switch variables==. <br>
[Concept: Variable system](?file=003-GC本体文档/0090-事件系统/002-变量)<br>


### Standalone Variable Operation
Obtain player variables:
> getVariable1<br> 
> Game.player.variable.getVariable(1);<br> 
> getStringVariable1<br> 
> Game.player.variable.getString(1);<br> 
> getSwitchVariable1<br> 
> Game. player.variable.getSwitch(1);<br>

Set player variables:
> setNumberVariable1: set to 100<br> 
> Game.player.variable.setVariable(1,100);<br> 
> setStringVariable1: set to "kds"<br> 
> Game.player.variable.setString(1, "kds");<br> 
> setNumberVariable1: set to 1<br> 
> Game.player.variable.setSwitch(1,1);<br>

Listening for changes in player variables (i.e. listening for changes in variables anywhere and adding logic here, e.g. numeric variable controls listen for changes in numeric variables and update the text to the latest variable value in real time).

Game.player.addListenerPlayerVariable(0, 1, Callback.New((typeID: number, varID: number, value: number) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// The value of type is 0 for numeric variable varID=1 for variable number 1 value for changed value<br>
}, this));<br><br>
// Listening for changes in string variable #1<br>
Game.player.addListenerPlayerVariable(1, 1, Callback.New((typeID: number, varID: number, value: string) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// The value of type is 1 for a string variable varID=1 for variable number 1 value for the changed value<br>
}, this));<br><br>
// Listening for changes in switch variable #1<br>
Game.player.addListenerPlayerVariable(1, 1, Callback.New((typeID: number, varID: number, value: number) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// The value of type is 2 for the switch variable varID=1 for the variable number 1 value for the changed value<br>
}, this));<br>

Access to the second week variables.
> Get the No. 1 Futurity value variable<br> 
> ClientWorld.variable.getVariable(1);<br> 
> Get the No. 1 Futurity string variable<br> 
> ClientWorld.variable.getString(1);<br> 
> Get the No. 1 Futurity switch variable<br> 
> ClientWorld.variable.getSwitch(1);<br>

Set up the second week variables.
> setNumberVariableNo.12Weekly: set to 100<br> 
> ClientWorld.variable.setVariable(1,100);<br> 
> setStringVariableNo.12Weekly: set to "kds"<br> 
> ClientWorld.variable.setString(1," kds");<br> 
> Set #1 diurnal value variable: set to 1<br> 
> ClientWorld.variable.setSwitch(1,1);<br>

Listening for changes in variables in the second cycle:

    // The value of type is 0 means the second week value variable varID=1 means variable number 1 value means the changed value<br>

    // The value of type is 1 means the second week string variable varID=1 means the variable number 1 value means the changed value<br>

    // The value of type is 2 means the second week switch variable varID=1 means the variable number 1 value means the changed value<br>
}, this));<br>


#### Network version variable operation
The network version is player variables and global variables (aka world variables). Player variables are per-player variables, while global variables are world-wide variables, common to all players. <br> 
Global variables have some overhead because they can be read and written in all threads and are accessed thread-wide at the bottom using locks, while variables are stored on the server side and are read or listened to by the client in a limited way. <br> 
Player variables can be listened to: that is, the server is asked to start listening to my player's N variable, and the server notifies the client to synchronize that variable when a change to that player's N variable occurs. <br> 
Whereas global variables can only get the current value at once and cannot be listened to for now. <br><br>

### Server Side
The server turns on the permission to allow access to global variables.<br>
>//Server settings to allow fetching of global value variable #1<br>
ServerWorld.setWorldVariableAccessible(0,1,true);<br>

The server modifies global variables.<br>
>//Set global value variable #1 to 100<br>
ServerWorld.setWorldVariable(1,100);<br>
//Set global string variable #1 to "kds"<br>
ServerWorld.setWorldString(1,"kds");<br>
//Set global switch variable #1 to on (1=on 0=off)<br>
ServerWorld.setWorldSwitch(1,1);

Server gets global variables.<br>
>//Get global value variable #1<br>
ServerWorld.getWorldVariable(1);<br>
>//Get global variable #1<br>
ServerWorld.getWorldString(1);<br>

# Code example: Calling the Event Library
The script in the event calls the event library of the event.<br>
>var cmdPage = Command.gameWorld.commonEventPages[Public event number];<br>
if(cmdPage)trigger.commandScope.push({ cmdPage: cmdPage, index: 0, commonEventID: Public event number, fragmentCommands: null })<br>

Calling independent public events in events:<br>
>var cmdPage = Command.gameWorld.commonEventPages[Public event number];<br>
var newFETrigger = new CommandTrigger(CommandTrigger.COMMAND_MAIN_TYPE_CALL_COMMON_EVENT, Public event number, trigger.scene, trigger.trigger, true, trigger.executor);
cmdPage.startTriggerEvent(newFETrigger, trigger.inputMessage);

Independent public events are called elsewhere:<br>
If it is a network version, you need to pre-set it to allow calls<br>
>GameCommand.startCommonCommand(Public event number);

# Code example: camera move
The camera jumps to the specified scene location (e.g. pixel coordinate point 500,500).<br>
>// First unlock the target, if the target is not locked, you can not call this code<br>
>Game.currentScene.camera.sceneObject = null;<br>
>// Then set the specified location<br>
>Game.currentScene.camera.viewPort.x = 500;<br>
>Game.currentScene.camera.viewPort.y = 500;<br>
>

Lens-bound scene objects:
>// The camera locks the scene object with ID=1 in the current scene (can be optimized to jog to his coordinates here first and then lock)<br>
// The binding is real-time following, if you want to create smooth following and other effects can not use the binding lens, but custom control lens coordinates.<br>
>Game.currentScene.camera.sceneObject = Game.currentScene.sceneObjects[1];<br>
