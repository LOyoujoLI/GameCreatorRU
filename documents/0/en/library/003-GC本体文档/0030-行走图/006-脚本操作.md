# Code example: avatar and standing avatar and StandAvatar
A avatar is an advanced display object that has action, direction, part, and sequence frame maps. <br> 
The origin of a avatar is the crosshair collimation in the editor. <br> 
A standing avatar is a subclass of a avatar, and can be thought of as a special avatar with no direction. <br>

## Creating a avatar
The avatar is generally not created separately, if on the scene is generally created scene object, the avatar as a sub-display object of the scene object exists.<br>
> // Create avatar #5<br>
> var a = new Avatar();<br>
> a.id = 5;<br>
> a.x = 300;<br>
> a.y = 300;<br>
> // Add to the stage<br>
>stage.addChild(a);<br>
> // Start playing<br>
> a.play();<br>

## Create a standing avatar
> // Create 3 standing avatar<br>
> var a = new StandAvatar();<br>
> a.id = 3;<br>
> a.x = 300;<br>
> a.y = 300;<br>
> // Add to the stage<br>
>stage.addChild(a);<br>
> // Start playing<br>
> a.play();<br>

## Listening for load completion events
>var a = new Avatar();<br>
>a.on(EventObject.LOADED,this,()=>{<br>
>&nbsp;&nbsp;&nbsp;&nbsp;// to do<br>
>});<br>
>a.id = 5;<br>

## Modify avatar/standing avatar attributes
> // Modify the face: face up (no direction in the standing avatar, no need to modify this attribute) <br>
> a.orientation = 8;<br>
> // Modify frame rate<br>
> a.fps = 20;<br>
> // Modify the action, based on the action ID<br>
> a.actionID = 3;
> // Skip to frame 5<br>
> a.setCurrentFrame(5,true);

## Operation of avatar part
Parts are managed by number-name unification, similar to the action list. For example 1 - head 2 - body.<br>
> // Add a part, ignore if the part already exists <br>
> a.addPartByID(Part number, avatar or standing avatar number)<br>
> // Replace a part <br>
a.changePartByAvatarID(avatar or standing avatar number, part number): boolean
 // Remove a part <br>
a.removePartByPartID(Part Number): boolean


## avatar component and standing avatar component
It is essentially a component that encapsulates the avatar and the standing avatar.
