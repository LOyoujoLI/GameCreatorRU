# Overview of avatar
A avatar is an advanced display object that has action, direction, part and sequence frame maps, and also supports avatar assistants.
> The current appearance of a avatar is determined by determining which action - which direction - which frame of the map: e.g. 3 - attack action -> down direction -> second frame<br>

### About the action
Unified management of action lists.<br>
You can customize a number of preset actions, such as standby, move, attack, dance, etc.<br>
>Of course, some templates may have fixed calls for some actions in the script, such as "Feather of cyan", which plays 2-move actions when moving. Different games or templates have different rules, and there is no fixed order or serial number, subject to the game item layer itself.

<img src="img/t28.png" /><br>

### About the direction
Currently GC supports 6 directions of avatar: 1 direction, 2 direction, 3 direction, 4 direction, 5 direction, 8 direction, where 1, 3, 5 direction will automatically flip horizontally in order to reduce material resources.
>For example, many games usually 2 directions are made by horizontal flip, but will store two directions of the map, if it is WEB side then it is equivalent to download an extra direction of the resource map.

In addition the GC generic direction terminology: direction system reference keypad with 5 as the center facing other numbers: 1 - lower left 2 - lower 3 - lower right 4 - left 6 - right 7 - upper left 8 - upper 9 - upper right.<br>
7 8 9<br>
4 5 6<br>
1 2 3<br>

### About the parts
Multiple parts can be loaded and the display hierarchy can be adjusted, and the parts play in the same direction, motion, and frame as the body.<br>
Use the parts list to unify management, such as 1 - head 2 - body 3 - weapon, etc., easy to change the call.<br>
>For example, in the "ARPG - Book of black" template the action of the protagonist falling to the ground with two smoke parts, so that multiple avatar reuse the two smoke parts.

### Merge Tool
If the clip is made up of multiple images, you can use the built-in merge tool to combine a collection of images in order to optimize loading speed and improve performance.<br>
<img src="img/new/3-1.png" />


### About the auxiliary body
Allows the author to customize the auxiliary body for use as an aid in calculating gameplay.<br>
==The same auxiliary body will be supported in different directions, movements and frames with different parameters, so that it can be used to assist in the calculation of precise movements in games such as combat.==<br>
==Multiple auxiliaries will be added soon: circles, triangles, polylines, and line segments.==<br>


<img src="img/t34.png" />




### Application Scenarios
1.The avatar can be used as a sub-display object of a scene object: for example, an NPC on the scene.<br>
2.The avatar can be displayed in the interface as a avatar component: for example, the avatar is displayed in the character panel.<br>
3.Used in scripts by the producer on any occasion: as an advanced display object that can be added under any display container at will.
