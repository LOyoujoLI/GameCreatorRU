# Overview of the scripting system
The software provides support for both Typescript and Javascript scripting languages, supporting editing of code for entire projects and inserting script code in places such as event directives. <br> 
==It also supports writing in VSCode == <br>

## About TypeScript
TypeScript can be thought of as an updated version of JavaScript that basically supports the JavaScript syntax and adds features that make it useful for developing medium to large projects. <br> 
Writing TypeScript within GC automatically compiles it to JavaScript at runtime and automatically sequences it based on library dependencies, inheritance relationships, etc., without installing additional configuration. <br>
<li>Added static types, classes, modules, interfaces and type annotations for code hinting and less prone to low-level errors caused by spelling</li> 
<li>Supported default argument values for functions</li> 
<li>Data requires explicit types and reports a red alert if it is obviously incompatible</li> 
<li>Introduced the concept of modules to encapsulate declarations, data functions and classes in modules</li> 
<li>Provides API files, import js and then add .d.ts or GC-specific .api files to use code hints</li>

## Code Editor
Use F11 to call out the code editor, allowing editing of the entire project's code. <br> 
Error reporting is output in the console, double click to lock to the line reporting the error. <br> 
Features:<br> 
> Quick compile: compile only modified TS files, usually fast and correct,<br> 
> Full compile: compile all TS files, usually needed after updating Git/Svn. <br> 
>Library project: support for additional dependent libraries, e.g. medium to large projects may have dozens of libraries,<br>
>Automatic sorting: sort by library dependency order, and within each library by priority js, then ts (TS will automatically sort by class inheritance)

Common shortcut key reference:<br>
>F9: Fast compile<br> 
F10: Full compile<br> 
CTRL+R: Find file<br> 
CTRL+SHIFT+F: Find code<br> 
ALT+LEFT Toggle to code before previous jump<br> 
ALT+RIGHT Toggle to code before next jump<br> 
CTRL+SHIFT+D Function variable comment<br> 
SHIFT+ALT+F Format code<br> 
CTRL+left mouse button Jump to code definition<br> 
CTRL+TAB Toggle selection file<br> 
CTRL+G Jump to line<br> 
F1 Open other shortcut list<br> 
ALT+/ Code completion<br>

## Developer Tools
F12 opens the developer tools to support breakpoints (you can also enter debugger in the code to add breakpoints, F10=no observation of steps within a function F11=observable steps within a function F8=skip).

## System generated code
Part of the system automatically generates some code, such as interfaces, scene object prototypes, data structures, etc. will automatically generate the relevant code. <br> 
Some of the code is generated as files: SceneObjectModels.ts and UIRuntime in the project Game/system folder, and some of it is generated for in-memory use. <br> 
The generated code can be called directly as well as being friendly to code hints.
> For example, the system settings screen in the "ARPG - Book of black" template is bound to a user-defined class called GUI_Setting. <br> 
> The tab component (typeTab) can be accessed and manipulated directly in the constructor of this class, and since the system automatically generates the relevant code, code completion and code hinting, etc. are also available and will be very convenient.

## The code inserted in the event
<br> The script will be compiled after the script is written on the event page in the editor, generating the corresponding JS executable. <br>
Client-side scripts for standalone events and server-side scripts for web-based events are pre-compiled when the game is running, so they are executed with the same efficiency as normal code,<br> 
and scripts in events can directly use events to call related variables (except for web-based clients):<br>
> commandPage:CommandPage; // Current event page<br>
> cmd:Command; // The current command, i.e. this script<br>
> trigger:CommandTrigger; // The current trigger (trigger line)<br>
> triggerPlayer:Player; // Trigger player (if any)<br>
