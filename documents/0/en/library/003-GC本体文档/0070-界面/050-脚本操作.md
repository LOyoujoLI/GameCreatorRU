# Code example: Interface
It is mainly used for various interfaces displayed in the game.<br>
[Concept: Interface](?file=003-GC本体文档/0070-界面/001-概述)<br>

## About the System Group Interface
System group interfaces are interfaces that are opened by events or using scripts (GameUI.show), are recorded in system preset groups, are unique, and can be managed by the ID of the interface. <br> 
> The underlying implementation, on the other hand, stores these created interfaces in an array, indexed by ID, so that interfaces that operate with the specified ID can find that interface. <br>

## Create interface (system group)
> // Create interface #3 (the base class is obtained using the GameUI.load method, here the type conversion,<br>
// (The default interface of No.3 is GUI_3 automatically generated by the system, you can also bind an interface class inherited from GUI_3)<br>
> var ui:GUI_3 = GameUI.load(3) as any;<br>
> // Add to interface layer<br>
> Game.layer.uiLayer.addChild(ui);<br>
> // Show the system group interface directly through GameUI.show (will be added to Game.layer.uiLayer, the method is the same effect as the above two lines of code)<br>
> var ui:GUI_3 = GameUI.show(3) as any;<br>


## Create interface (non-system group)
>// Created using GameUI method (clone mode)<br>
var ui:GUI_3 = GameUI.load(3,true) as any;<br>
// New directly, GUI_3 is the system automatically generated interface class<br>
var ui = new GUI_3();<br>

## Access to the controls of the interface
The interface auto-created class automatically creates the variable based on the name of the control for direct access to the control.<br>
If the interface controls are renamed then only the first one is recorded, but you can manipulate the API to get all the child display objects with the specified name.<br>
> // For example, if there is a button with the name btn on screen 3, then it can be accessed directly like this.<br>
> var ui = new GUI_3;<br>
> ui.btn.x = 123;<br>

If the next layer of an interface has 3 sub-objects named btn, get all the sub-objects named btn and store them in btns
> var ui = new GUI_3;<br>
> var btns = [];<br>
> for(var i=0;i< ui.numChildren;i++){<br>
> &nbsp;&nbsp;&nbsp;var comp = ui.getChildAt(i);<br>
> &nbsp;&nbsp;&nbsp;if(comp.name=="btn")btns.push(comp);<br>
> }<br>

## Interface binding class
Since the system automatically generates GUI_XX based on the interface, you can customize the class to inherit from GUI_XX, assuming that there is a button named btn in the interface, you can access the operation directly in the constructor.
>class MyGUI extends GUI_3 {<br>
>&nbsp;&nbsp;&nbsp;constructor() {<br>
>  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;super();<br>
>  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;this.btn.x = 500;<br>
>  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// Let the control named btn register the click event and execute the onClick function when it is clicked<br>
>  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;this.btn.on(EventObject.CLICK,this,this.onClick);<br>
>&nbsp;&nbsp;&nbsp;}<br>
&nbsp;&nbsp;&nbsp;private onClick(e:EventObject):void{<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;alert("You clicked on me");<br>
&nbsp;&nbsp;&nbsp;}<br>
>}<br>
