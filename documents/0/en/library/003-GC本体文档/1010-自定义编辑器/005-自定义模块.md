# Customized Modules
The custom module can be used in the database after it is created and can be considered as a custom data editor.<br>
> For example, the characters, professions, skills, equipment, props, and states in the "ARPG - Book of black" are all custom modules pre-written by the template

<img src="img/t65.png" />

## Open the Custom Module Editor
Menu -> Custom Editor -> Custom Modules (ALT+3)<br> The Custom Module Editor allows you to visualize the operator panel for making modules, including fixed components, components that appear only when certain conditions are met, or blocks.

## Create custom modules
Definition modules are managed uniformly using numbering.<BR>
Adjust the range of available modules by changing the maximum value.<BR>

## The interface of the editing module
The methods for making the editor interface visually in the custom editor are the same, refer to [[Custom Editor Panel](?file=003-GC本体文档/1010-自定义编辑器/003-自定义编辑面板)]。

## Visualization of production module data
Open the database - select the module to add or delete data under the module, and operate in the same way as other modules (e.g. interface modules)<br> 
If you have created and designed a module called props, then you have to start filling in the data, 1 - red drugs 2 - sapphires 3 - branches, etc. <br> 
This way of making data will be more friendly and WYSIWYG because of the pre-designed effect of the components used, the range of values taken, the preview, etc.

## Automatic system generation of API
The system automatically generates a CustomModuleRuntime.api file based on all the current custom modules,<br> 
which is only used in the current memory and not the actual hard disk file, and the API is not available if you use a third-party IDE. <br> 
The corresponding module class is accessed by using ==Module\_Module_Name==, and the system automatically generates the class definition based on predefined variables and their types, default values, etc. <br> 
For example, if I create a module named Props, then the system generates Module_Props , == the code class is only used for code hints:== <br>

## Creating data within a module
Use GameData.newModuleData(moduleID,dataID,whether copy) to create a pre-defined data under that module,<br> 
e.g. create data 2-sapphire in module 1-props module: GameData.newModuleData(1,2)<br>
Reference API:[GameData.api](?file=006-API-2D单机内核/001-通用/017-GameData)<br>
<img src="img/2/自定义/9.png" />

## Get the system pre-defined data model of custom modules
Use GameData.getModuleData(ModuleID,DataID) to get the pre-defined data in the database. <br>
For example, in some cases you don't need to create a new data, but just want to get what the preset value is. <br>
Reference API:[GameData.api](?file=006-API-2D单机内核/001-通用/017-GameData)<br>
<img src="img/2/自定义/10.png" />

## Get the total length of data for the specified module
Use GameData.getModuleData(ModuleID,Category) to get the total number of data pre-defined in the database. <br> 
For example, make a store that lists all items and prices, which can be displayed by getting the total length and traversing through to generate props:<br>
Reference API:[GameData.api](?file=006-API-2D单机内核/001-通用/017-GameData)<br>

## Implementing the logic functions of the module
What exactly the module is used for depends on the design of the creator, the custom module system is only used to design the necessary attribute parameter panels for the module and to fill in the data. <br> 
For example, if the prop module is eventually used to load into the player's backpack, then a backpack attribute would be designed in [player data](?file=003-GC本体文档/1010-自定义编辑器/018-玩家属性编辑), and the player's backpack would be loaded with multiple prop data.
