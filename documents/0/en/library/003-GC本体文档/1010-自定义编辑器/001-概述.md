# Custom System Editor Overview
Custom editors are used to modify and extend the editor itself. <br> 
>To make it easy to extend the editor and create an editor exclusive to game templates, GC provides a complete custom editor solution. <br> 
>The game templates we usually see are a complete solution written using the system module + custom editor + scripter. <br> 
>Also the plugin can carry the associated editor interface for others to use with their own visual production interface. <br>

GameCreator offers a variety of custom editors:<br>

## Visualize the creation of custom editor panels
Allows users to visually build editor panels to extend the game editor. <br> 
The editor created visually automatically generates the corresponding code or API for direct access to call or implement the corresponding functionality. <br> 
-- Visualize the creation of properties and the corresponding controls<br> 
-- Visualize the writing of dependency appearance conditions and support for scripting conditions (e.g. component B needs to appear when component A has a value of 3)<br> 
-- ==In the future there will be support for writing editor panels in a fully programmatic form, providing support for more free editor panel creation (e.g. making panels like the interface editor)== <br> 
And the editor panel filled with values represents the value of the editor. br> And the meaning of the values filled in the editor panel is determined by the creator, and the script is used to implement its functionality (with some support for pure event implementation).

## Custom module
Custom modules can be used in the database once they are made and can be seen as custom data editors. <br> 
>For example, characters, professions, skills, equipment, props, and stats in "ARPG - Book of black" are all custom modules pre-written by the template

<img src="img/t65.png" />

## Custom Data Structures
Custom data structures are generally used in conjunction with custom attributes, some of which are not of a single type. <br> 
For example, a player's backpack records information about props, numbers, and equipment that the player has, rather than referring to a single value, and such compound cases can be made using custom data structures. 
> Similar to attribute types: Object instead of primitive number, string, boolean types

## Custom World Settings
The world is the only world in the game, and the world is the world common to all players (there is only one player in the single player kernel). <br> 
The world properties are edited through the visual editor, and the world properties are implemented through scripts or events so that this game template can visually adjust the relevant world parameters. <br> 
>For example, the "RPG - Feather of cyan" template has preset parameters such as sound effects, events at game start, etc. so that the author can visually adjust these parameters

<img src="img/t10.png" /><br>

## Custom Player Settings
Players are in-game players, each of which corresponds to a player identity (there is only one player in the standalone kernel). <br> 
The player attributes are edited through the visual editor and implemented through scripting,<br> 
so that this game template can visually adjust the relevant player parameters, and each player has these attributes after setting. <br> 
>For example, the "ARPG - Book of black" template has preset gold and backpack attributes, and the player has the gold and backpack attributes.<br> 
The reason why these attributes are not system-installed but are custom attributes is because not every game has these settings for gold and backpack, and different games have different settings.

<img src="img/t11.png" /><br>

## Custom Scene Object Prototype
Scene object prototypes are a predefined class of objects that have == custom properties== and == custom display objects== and == binding scripts==. <br> 
Defining scene object prototypes is typically made by senior developers, as it generally involves code to implement the relevant functionality. <br> 
>For example, two prototypes are defined for animals and rocks. Animals generally have the ability to live, move, etc., while rocks do not. <br> 
>We write animal-related features for animals and rock-related features for rocks, so that both animals and rocks can exist on the scene

## Custom Conditional Divergence
In addition to the several condition divergence categories given by the system, custom condition divergence is also supported. <br> 
Different items (templates) may have different custom conditional divergences. <br> 
> For example, several custom conditional divergences are provided in "RPG - Feather of cyan" to make more determinations related to that template

<img src="img/2/自定义/2.png" /><br>

## Custom Game Values
Custom game values are generally used to make values related to that game (template). <br> 
For example, the number of gold coins a player has. <br> 
Different items (templates) may have different custom game values. <br> 
The value can be used to record it using variables, or to judge it, or to display it directly in the interface [[Game Value Component](?file=003-GC本体文档/0070-界面/009-游戏数值组件)]. <br> 
From a programmatic point of view, a custom game value is equivalent to a function that can take in parameters and return a numeric value as a result. <br> 
> For example, "RPG - Feather of cyan" provides some custom game values to get, determine, and display

<img src="img/2/自定义/3.png" /><br>

## Custom game string
Custom game strings are generally used to make strings related to that game (template). <br> 
Different projects (templates) may have different custom game strings. <br> 
The strings can be used to record with variables, or to judge, or to display directly in the interface [[Game String Component](?file=003-GC本体文档/0070-界面/010-游戏字符串组件)]. <br> 
From a programmatic perspective, a custom game string is equivalent to a function that can take in parameters and return a string as the result. <br>


## Custom Object Module
Scene object modules are assemblable components, and when a scene object has a particular module, it has the characteristics of that module. <br> 
Contains properties, display objects, bound scripts, editor preview


## Custom Materials
The material system is used to add "material effects" to the displayed object, such as tint, blur, distortion, etc. <br> 
Materials can consume more performance, so please use them wisely according to the situation. <br> 
See [Material](?file=003-GC本体文档/0100-材质系统/001-概述) for more information.
