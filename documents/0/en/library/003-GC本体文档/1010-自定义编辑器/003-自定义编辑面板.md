# Custom Edit Panel
The custom editor panel allows users to visually build editor panels to extend the game editor. <br> 
The editor created visually automatically generates the corresponding code or API for direct access to call or implement the corresponding functionality. <br> 
-- Visualize the creation of properties and the corresponding controls<br> 
-- Visualize the writing of dependency appearance conditions and support for scripting conditions (e.g. component B needs to appear when component A has a value of 3)<br> 
-- ==In the future, editor panels will be written in a fully programmatic form, providing support for more free-form editor panel creation (e.g. creating panels like interface editors)== < br> And the editor panel fills in the code and API to extend the game editor. <br> 
And the meaning of the values filled in the editor panel is determined by the creator, and the script is used to implement its functionality (with some support for pure event implementation).

### Dependency condition
Dependent condition means that the decision to change the component or block to be displayed is based on the value of the existing component. <br> 
Currently, the decision to display or not is based on the value of the dropdown and checkboxes in the component under this data. <br> 
If multiple conditions are added then all conditions must be met before they are allowed to appear:<br> 
Normal conditions: use the simple current value of the component to make the determination. <br> 
Expression conditions: Use script editing to support arbitrary expressions. 
> Write conditional expressions based on attribute name values (currently supports checkboxes and checkboxes), e.g. (attrs.xxx>5 && attrs.yyy<=10) || attrs.zzz==6;
