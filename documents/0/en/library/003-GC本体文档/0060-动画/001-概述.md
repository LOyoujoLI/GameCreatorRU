# Animation System Overview
The author can pre-set the animation in the animation editor for use in the game by using a series of operations such as ==Photo Gallery==, various ==Animation Layers==, ==K Frames==, etc.

<img src="img/t36.png" />

## Sequence frame animation
### Photo Gallery
You can import pictures as gallery material for animation, and you can also cut pictures during the import process. Dragging the imported picture material to the view area will add a new animation layer: ==picture layer==.<br>
<img src="img/t38.png" /><br>
<img src="img/t37.png" />


### Animation layers
Animation layers are divided into: ==sound effects layer==, ==picture layer==, ==animation layer==.<br>
>[Sound layer]: represents a sound effect, playing a sound effect from the start keyframe. <br> 
>[Picture layer]: represents a picture, displayed from the start keyframe<br> 
>[Animation layer]: represents another animation nested (may include particles)

### Special layer: Target effect layer
The animation supports acting on a certain target and part of the animation layer is at the bottom of the target and part of the animation layer is in front of the target, while the effect of the target also supports K-frames.<br>
>For example, to create an aura that is obscured by the target, and an explosion effect that obscures the target while producing a reddish effect on the target.

<img src="img/t39.png" />



### K-frame
Each animation layer can be keyframed, with automatic transitions between keyframes. The default is a linear transition, but you can also set a variety of preset non-linear transitions or custom curves.

<img src="img/t40.png" />




## Concept: Particle animation
Particle animation is another form of animation, different from K-frame animation, which is a kind of animation that continuously generates particles out.<br>
A particle animation can be considered as an animation with only one particle jet, if you want to make multiple jets or adjust the orientation etc. you can attach it to the normal animation.

<img src="img/t41.png" />
<img src="img/t43.png" />

### Preset configuration
Support the author to save the current particle configuration as a preset and switch to the preset configuration.<br>
<img src="img/t42.png" />


## Application Scenarios
1.The animation can be used as a sub-display object of the scene object: for example, a pre-defined mount animation.<br>
2.The scene object has the ability to play an animation with the target being the avatar of the scene object.<br>
3.Use animation in image-related commands to create various effects, usually for animation effects relative to the screen.<br>
4.Placement of animations in the interface.<br>
5.Script to use animation as an advanced display object at will.
