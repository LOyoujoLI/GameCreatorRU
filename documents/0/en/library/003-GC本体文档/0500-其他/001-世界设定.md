# World Setting
The world is the only world in the game, and the world is common to all players (there is only one player in the single player kernel).<br>

### Custom world attributes.
> The advanced author edits the world properties through the visual editor and implements the world properties through scripting so that the game template can visually adjust the relevant world parameters. <br>
>For example, the "ARPG - Book of black" template has preset parameters such as title music, end game music, etc. so that the author can visually adjust these parameters

See [Custom World Settings](?file=003-GC本体文档/1010-自定义编辑器/015-世界属性编辑) for more。

### Data storage.
Custom world attributes and global variables/diegetic variables are encapsulated ==system automatically stored==, the network version kernel is stored to the database periodically, while the standalone version is stored when the archive-related functions are invoked. 
## Script Call: World Properties
### Stand-alone version - World data reading and writing.<br>
>For example, if you define a world variable named abc, which is of type numeric<br>
>// Get World Attributes<br>
>WorldData.abc;<br>
>// Modify world attributes<br>
>WorldData.abc = 123;<br>

### Network version - Client world data reading.<br>
<br>The client-side cached value is old if the world attributes are not manually synced to the player after the player logs in. <br> 
Also, although the client can modify the value, it is not really meaningful because the data is stored on the server. <br>
>//Get the client-side cache of world attributes: for example, a world variable named abc is defined, which is of numeric type<br>
> WorldData.abc<br>

### Network version - server-side world data reading and writing.<br>
Read world custom data:<br>
>WorldData.abc<br>

Writing world custom data: Since world data needs to be synchronized across the scene threads, special functions need to be called in order to synchronize it. <br> 
Using ServerWorld's dataWrite method, caveats: <br> 
-- The code within the method runs independently in other threads, so it cannot rely on external variables, and the need to use external data should be passed in via args. <br> 
-- The method should not contain random elements within the method, it must be deterministic, otherwise there is a synchronization error. <br> 
-- Arguments must be JSON packable copied as pure data. <br>
Example:<br
>var a = new DataStructure data structure2();<br>
a.AAA = MathUtils.rand(1000000000) + 1000000000;<br>
a.BBB = 22222222<br>
a.CCC = 33333333<br>
var aaa = Math.random();<br>
ServerWorld.dataWrite(function (a: DataStructure data structure2,aaa) {<br>
&nbsp;&nbsp;&nbsp;&nbsp;ServerWorld.data.(My Data Group).push(a);<br>
&nbsp;&nbsp;&nbsp;&nbsp;ServerWorld.data.(Random Number)=aaa;<br>
}, [a,aaa]);
