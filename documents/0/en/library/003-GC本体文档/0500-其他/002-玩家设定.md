# Player Settings
Players are the in-game players, each corresponding to a player identity (there is only one player in the standalone version). <br> 
Players have ==player custom attributes== and ==player variables== and ==player scenario objects==.

### Player's custom attributes.
> The advanced author edits player attributes through the visual editor and implements player attributes through scripting,<br> 
so that the game template can visually adjust the relevant player parameters, and each player has those attributes after setting. <br> >
For example, the "ARPG - Book of black" template has preset gold and backpack attributes, and the player has the gold and backpack attributes.<br> 
The reason these attributes are not system-installed but are custom attributes is because not every game has gold and backpack settings, and different games have different settings.

Please refer to [Custom Player Settings](?file=003-GC本体文档/1010-自定义编辑器/018-玩家属性编辑) for more. <br>

### Player's scenario object:
Each player has one == scenario object== by default. 
> Although the default player has only one scene object data, it can be logically written to have multiple players or control multiple scene objects, such as "ARPG - Book of black" is <br> 
an ARPG game template with multiple players, other players are created scene objects when they join, and these objects will be brought to the new scene when switching scenes. <br> 
When storing, the system automatically records the scene object of the current scene, so you only need to record additional key data (such as the scene object ID of your teammates).

Also have this scene object corresponding to the == scene object prototype == preset == custom scene object properties == and == custom display objects==. 
>For example, in RPG, usually both the player's scenario object and the NPC have a avatar, which is a custom display object, and may also have custom attributes like movement speed, <br> which are pre-defined models via the ==Scene Object Prototype==, although there is support for setting many different ==Scene Object Prototypes==.


### Data storage:
Custom player attributes and player variables are encapsulated == automatically stored by the system==, the network version kernel is stored to the database periodically, while the standalone version is stored when the archive-related functions are invoked.

## Script Call: Player Properties
#### Stand-alone version - Player data reading and writing.<br>
Since the single player version has only one player, it can be accessed through Game.player.
> // Read player-defined attributes:<br>
> Game.player.data.(My Custom Properties 1)<br>
> // Operating player-defined attributes:<br>
> Game.player.data.(My Custom Properties 1)=123456;<br>

#### Network version - Client player data reading.<br>
The player-defined data of the network version of the client comes from the synchronization of the server (the synchronization method is determined by the project layer, which properties can be synchronized can be pre-defined in the editor).<br>
> Get my player's attributes<br>
> Game.player.data.(My Custom Properties 1)<br>
> Get the attributes of other players: for example, the attributes of the player corresponding to object #10 on the scene<br>
> var so = Game.currentScene.sceneObjects[10];<br>
> // Determine if this object is a player<br>
> if(so.player){<br>
>&nbsp;&nbsp;&nbsp;&nbsp;trace(so.player.data.(My Custom Properties 1));<br>
> }

#### Network version-server player data read/write.<br>
If you need to manipulate the data of an offline player or a player not in the current thread, you can use the methods provided by the system or implement the corresponding logic yourself.<br>
> // Read the player's properties (player is an instance or subclass of the SeverPlayer class)<br>
> player.data.(My Custom Properties 1)<br>
> // Write to player attributes<br>
> player.data.(My Custom Properties 1) = 123;<br>
> // Write data to a player not in the current thread or to an offline player (i.e. the player cannot be found in the current thread, but you want to manipulate the data written to that player)<br>
> Refer to the operationPlayerData method of [ServerPlayer](?file=007-API-2D网络内核/004-服务端/013-ServerPlayer).<br>
