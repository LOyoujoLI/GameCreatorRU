# Code Example: Calling a Public Event
Scripts in events call public events<br>
>var cmdPage = Command.gameWorld.commonEventPages[public event number];<br>
if(cmdPage)trigger.commandScope.push({ cmdPage: cmdPage, index: 0, commonEventID: Public event number, fragmentCommands: null })<br>

Invoking Standalone Public Events in Events<br>
>var cmdPage = Command.gameWorld.commonEventPages[public event number];<br>
var newFETrigger = new CommandTrigger(CommandTrigger.COMMAND_MAIN_TYPE_CALL_COMMON_EVENT, public event number, trigger.scene, trigger.trigger, true, trigger.executor); cmdPage.startTriggerEvent(newFETrigger, trigger.inputMessage);

independent public event called elsewhere<br>
If it is online, you need to set up in advance to allow calls<br>
>GameCommand.startCommonCommand(public event number);

# Code Sample: Location Move

EventUtils.happen(triggerPlayer.sceneObject, Command.EVENT_SYSTEM_COMMAND_START, [2]);<br>
//Mobile site<br>
triggerPlayer.toScene(scene number, scene pixel horizontal coordinate, scene pixel vertical coordinate);<br>

# Code Example: Camera Move
The camera jumps to the specified scene location (e.g., pixel coordinates point 500,500):<br>
>//Unlock the target first. If the target is not locked, you can not call this code<br>
>Game.currentScene.camera.sceneObject = null;<br>
>//Set the specified position again<br>
>Game.currentScene.camera.viewPort.x = 500;<br>
>Game.currentScene.camera.viewPort.y = 500;<br>
>

lens binding scene objects
>//The camera locks the scene object with ID=1 in the current scene (it can be optimized to slow down to his coordinates before locking)<br>
//This binding is real-time following. If you want to make smooth following effect, you can not use binding lens, but customize control lens coordinates. <br>
>Game.currentScene.camera.sceneObject = Game.currentScene.sceneObjects[1];<br>
