
# code example: Manipulate variables
Variable system is the system default data array, the system provides the == global variables (two weeks)== and == player variables == two forms of variables. <br>
And each variable according to the variable type is divided into: == numeric variable ==,== string variable ==,== switch variable ==<br>

### Standalone Variable Actions
Get Player Variables
> Get value variable #1<br>
> Game.player.variable.getVariable(1);<br>
> Get string variable #1<br>
> Game.player.variable.getString(1);<br>
> Get switch variable #1<br>
> Game.player.variable.getSwitch(1);<br>

Set Player Variables
> Set value Variable #1: Set to 100<br>
> Game.player.variable.setVariable(1,100);<br>
> Set string variable #1: set to "kds"<br>

#### Network Version Variable Operation
The network version is player variables and global variables (aka world variables). Player variables are per player variables, while global variables belong to the entire world and are common to all players.<br>
Global variable can be read and written by all threads, and are access by all threads in that form of locks at the bottom layer, so that certain overhead is realize,
The variable is stored in the server, the client limited reading or listening. <br>
Player variables can be monitored: that is, the server is requested to start monitoring the N variable of the player, and the server will notify the client to synchronize the variable when the N variable of the player changes. <br>
And global variables can only one-time access to the current value, temporarily can't listen. <br><br>

### Server Side
The server turns on the permission to allow access to global variables:<br>
>//Server setting to allow fetching of global value variable #1<br>
ServerWorld.setWorldVariableAccessible(0,1,true);<br>

The server modifies global variables:<br>
>/Set global value variable #1 to 100<br>
