# Code example: animation and particles
The author can pre-set the animation in the animation editor for use in the game by using a series of operations such as ==Photo Gallery==, various ==Animation Layers==, ==K Frames==, etc.<br>
Particle animation is another form of animation, different from K-frame animation, which is a kind of animation that continuously generates particles out.<br>
[Concept: Animation](?file=003-GC本体文档/0060-动画/001-概述)<br>

## Create animation
> // Create animation #5<br>
> var a = new Animation();<br>
> a.id = 5;<br>
> a.x = 300;<br>
> a.y = 300;<br>
> // Add to the stage<br>
>stage.addChild(a);<br>
> // Start playing<br>
> a.play();<br>

## Listening for load completion events
>var a = new Animation();<br>
>a.on(EventObject.LOADED,this,()=>{<br>
>&nbsp;&nbsp;&nbsp;&nbsp;// to do<br>
>});<br>
>a.id = 5;<br>

## Acts on the target
Add to the display object, while part of the animation layer can be displayed below the target and part of the animation layer above the target, and support target effects
>// Create a avatar object (for use as a target)<br>
>var avatar = new Avatar();<br>
>avatar.id = 1;<br>
>avatar.actionID = 2;<br>
>avatar.play();<br>
>// Creating the Underlay<br>
>var lowLayer = new GameSprite();<br>
>// Create high level<br>
>var highLayer = new GameSprite();<br>
>// Create container for loading Avatar<br>
>var root = new GameSprite();<br>
>root.x = 500;<br>
>root.y = 500;<br>
>// Stuff the bottom, Avatar, and top layers into separate containers<br>
>root.addChild(lowLayer);<br>
>root.addChild(avatar);<br>
>root.addChild(highLayer);<br>
>// Add to the stage<br>
>stage.addChild(root);<br>
>// Create animations and attach them to the target, so that the target effect can be played, or part of the level can be displayed at the bottom of the target and part of the level at the top of the target<br>
>var ani = new Animation();<br>
>ani.id = 1;<br>
>ani.loop = true;<br>
>ani.addToGameSprite(avatar, lowLayer, highLayer);<br>
>ani.play();
