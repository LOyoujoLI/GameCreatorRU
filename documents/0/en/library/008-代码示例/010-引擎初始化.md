# Code example: Engine initialization
The GameCreator engine does not contain any gameplay logic, which needs to be implemented in the project layer.<br>
The game template is a project that uses the GameCreator engine to implement the gameplay and the editor to create the corresponding relevant editor panel.
> Open F11 to enter the script editor and start editing the script

# Stand-alone version
1.Create global variable: Game, as an instance of GameBase or its subclasses.<br>
>var Game: GameBase = new GameBase();

2.Listen to the engine initialization completion: use EventUtils event tool class to listen to the engine initialization completion event<br>

>EventUtils.addEventListener(ClientWorld, ClientWorld.EVENT_INITED, Callback.New(function () {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// Write arbitrary logic <br>
>}, this), true);

3.New game: After calling "new game", the system will send the event of entering the scene, listen to the event of entering the scene, and then write the logic:<br>
>// First listen to the event of entering the scene: new game, read file, switch scene<br>
// @param sceneModelID Scene Model ID<br>
// @param state Status 0-Switch game scenario 1-New game 2-Read archive<br>
EventUtils.addEventListener(ClientScene, ClientScene.EVENT_IN_NEW_SCENE, Callback.New((sceneModelID: number, state: number) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// Create scenes, scene objects, etc. Logic<br>
}, this));<br>
// Calling the functions provided by the engine: new games<br>
SinglePlayerGame.newGame();


4.Read archive: Call the function provided by the engine: "Read archive", after the system will send the event of entering the scene, listen to the event of entering the scene and then write the logic. (If there is an existing game archive, it can be read successfully)
>// First listen to the event of entering the scene: new game, read file, switch scene<br>
// @param sceneModelID Scene Model ID<br>
// @param state Status 0-Switch game scenario 1-New game 2-Read archive<br>
EventUtils.addEventListener(ClientScene, ClientScene.EVENT_IN_NEW_SCENE, Callback.New((sceneModelID: number, state: number) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// Create scenes, scene objects, etc. Logic<br>
}, this));<br>
// Calling the functions provided by the engine: new games<br>
SinglePlayerGame.loadGame(1, Callback.New((success: boolean) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;if(!success){alert("Failed to read archive");return;}<br>
}, this));

Usually new games or reading archives are invoked by clicking on the interface button.<br>
Some casual mini-games do not require the creation of new games and read archives, so feel free to write arbitrary logic after the engine initialization.<br>

# Network version
### Client
1.Create global variable: Game, as an instance of GameBase or its subclasses.<br>
>var Game: GameBase = new GameBase();

2.Listen to the engine initialization completion: use EventUtils event tool class to listen to the engine initialization completion event<br>

>EventUtils.addEventListener(ClientWorld, ClientWorld.EVENT_INITED, Callback.New(function () {<br>
&nbsp;&nbsp;&nbsp;&nbsp;// Write arbitrary logic <br>
>}, this), true);

3.Connecting to the server (the default server and port can be set in the editor's general configuration; for code changes, refer to[Config.api](?file=007-API-2D网络内核/001-通用/015-Config)）<br>
&nbsp;&nbsp;&nbsp;The completion server must be started first, otherwise the connection will fail<br>
>// Generate random keys and UIDs to simulate anonymous players<br>
Game.playerKey = Math.floor(Math.random() * 20000 + 20) + "_asdasdasd";<br>
// ClientScene.EVENT_IN_NEW_SCENE event will be triggered to enter the specified scene after successful login.<br>
ClientWorld.startLogin(Callback.New(function () {<br>
&nbsp;&nbsp;&nbsp;&nbsp;trace("Connected successfully, logged into the game, my UID=", Game.player.uid);<br>
}, this), Callback.New(function () {<br>
&nbsp;&nbsp;&nbsp;&nbsp;alert("Network Error");<br>
&nbsp;&nbsp;&nbsp;&nbsp;window.location.reload();<br>
}, this));

4.When the connection is successful, the system will send an event requesting to log in to the scene, and when the event is received, you can create the scene.<br>
When finished the server can be notified and then the server and client start to execute the normal communication logic.<br>
>// sceneModelID Indicates the scene ID to be logged in<br>
EventUtils.addEventListener(ClientScene, ClientScene.EVENT_IN_NEW_SCENE, Callback.New((sceneModelID: number) => {<br>
&nbsp;&nbsp;&nbsp;&nbsp;Logic for creating scenarios XXX<br>
&nbsp;&nbsp;&nbsp;&nbsp;// The communication logic is customized by the project: for example, the BB method of the AA class of the server is called remotely with parameters [1,2,3], and the server needs to set permission to allow the method to be called<br>
&nbsp;&nbsp;&nbsp;&nbsp;ClientMsgSender.rpc("AA", "BB",[1,2,3]);<br>
}, this));<br>

### Server Side
1.The class GameServerPlayer must exist, the underlying layer will create a player class of that name, so the class must exist.<br>
&nbsp;&nbsp;It mainly facilitates the writing of code hints and player class logic at the project level.
>/\*\*<br>
&nbsp;\* Project layer player class<br>
&nbsp;\* The underlying layer will create the player class with that name, so the class must exist.<br>
&nbsp;\* Created by 黑暗之神KDS on 2020-03-03 10:01:38.<br>
&nbsp;\*/<br>
class GameServerPlayer extends ServerPlayer {<br>
&nbsp;&nbsp;&nbsp;&nbsp;/\*\*<br>
&nbsp;&nbsp;&nbsp;&nbsp;\* Constructors<br>
&nbsp;&nbsp;&nbsp;&nbsp;\* @param key Player unique key<br>
&nbsp;&nbsp;&nbsp;&nbsp;\* @param uid Player unique uid<br>
&nbsp;&nbsp;&nbsp;&nbsp;\*/<br>
&nbsp;&nbsp;&nbsp;constructor(key: string, uid: number) {<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;super(key, uid);<br>
&nbsp;&nbsp;&nbsp;}<br>
}<br>

2.Create scenario binding classes in order to implement related functions, such as:
>class GameServerScene extends ServerScene<br>
