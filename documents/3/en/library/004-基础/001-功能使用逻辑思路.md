## Getting Started

Inevitably in making a game we will have a situation where we want to do a certain function but are not sure how to start, so here are the main ideas about function calls.

## General game settings

### World settings

Most of the game's general settings are stored in **World Settings**, so if you want to change the general settings you can look at **World Settings**.

![](img/Basic/001/001.png=600-)

### Player Settings

The player settings store the settings related to the player object within the player settings.

![](img/Basic/001/002.png=600-)

### Common Event

**Common Event->System Events** Also stores settings made via commands, and the associated trigger event pages.

![](img/Basic/001/003.png=600-)

## Avatar

We can find the **Avatar** module in the **Database** and just add the corresponding material directly.

Avatar, also called sprite. It is used to control the style of players and NPCs.

![](img/Basic/001/004.png=600-)

## Standing avatar

We can find the **Standing avatar** module in the **Database** and add the corresponding material directly.

Subsequent templates related to the standing avatar are called through the standing avatar module.

![](img/Basic/001/005.png=600-)

## Animation

We can find the **Animation** module in the **Database** and just make it as needed afterwards.

Subsequent templates related to animation are called through the animation module.

![](img/Basic/001/006.png=600-)

## Dialog box styles

We can find the **Dialog Styles** module in the **Database** and just replace the corresponding material directly.

If there are individual components you don't want to display, you can drag them off-screen or set the transparency to 0.

The styles called by the displayed dialogs are based on this.

![](img/Basic/001/007.png=600-)

## Game interface

We can find the **Interface** module in the **Database** and directly add the corresponding components as well as materials.

All the interfaces in the game can be made by the interface module, such as the title interface.

![](img/Basic/001/008.png=600-)

## item

We can find the **item** module in the **Database** and just add the material directly.

The item in the game are set by this module.

![](img/Basic/001/009.png=600-)

## Get the content of the game

There is a game variable in the variables, we can get the content of the game through the game variable.

## Game system process

Starting the game, archiving, and returning to the title screen are all controlled by commands.
